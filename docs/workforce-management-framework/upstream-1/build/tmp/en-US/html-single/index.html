Chapter 1. Introducing RainCatcher

1.1. Understanding RainCatcher

1.2. RainCatcher Overview

RainCatcher is a software framework and is made up with varying node.js modules which are written in TypeScript. These moudles are packaged and distributed as part of the RainCatcher solution. The primary use of the RainCatcher solution is to provide a Workforce Management Oriented framework for Workforce Management applications.

The RainCatcher solution comprises of two fully independent frameworks:

  • Workflow Management Framework
  • Rapid Application Development Framework

1.3. Rapid Application Development Framework

The RainCatcher Rapid Application Development Framework allows to build production-grade Node.js and Cordova based applications.

1.4. RainCatcher Demo Application

RainCatcher contains a demo application which contains many of the RainCatcher features. The demo contains:

  • Implementations for a Workflow Management Solutions
  • Website, Mobile and server demo applications serving as reference implementations for the RainCatcher framework

1.5.1. Module structure

RainCatcher modules are written in TypeScript.

The main files are modules that by default export a class containing the main implementation for the module’s intent,with a named export containing a public interface that should be depended upon, and reimplemented. 

import Implementation, { PublicInterface } from '@raincatcher/module';

1.5.2. Default exports

Each module exports by default an implementation and set of interfaces by name. Default implementation will usually implement most common use case for module. Along with the default export, all modules include a public interface definition that are depended upon by other RainCatcher modules instead of depending on the implementation itself.

1.6. Introducing RainCatcher Features

1.6.1. Logging Module

The RainCatcher Logger Module is a logging facade for node.js based applications and it is used by all RainCatcher modules. The RainCatcher Logger Module is is contained in the raincatcher-core module.

There are two implementations of the RainCatcher Logger module, Bunyan and Console. By default, the Bunyan and the Console implementations are not setup and so, no logging exists by default. Users can use their own loggers by wrapping their logger into the provided class.

For more information, see the Logger Example.

4.1. Using RainCatcher Logger and Setting a Logging Level

4.1.1. Logging Module

The RainCatcher Logger Module is a logging facade for node.js based applications and it is used by all RainCatcher modules. The RainCatcher Logger Module is is contained in the raincatcher-core module.

There are two implementations of the RainCatcher Logger module, Bunyan and Console. By default, the Bunyan and the Console implementations are not setup and so, no logging exists by default. Users can use their own loggers by wrapping their logger into the provided class.

For more information, see the Logger Example.

4.1.2. Procedure

4.1.2.1. Importing the Bunyan Logging Module

To incorporate the Bunyan logging into a project, the RainCatcher Logging Module is imported and then the Bunyan Logging Module is initialized.

Prerequisites

  • raincatcher-core module must be installed.

Procedure

  1. Add the following code to the file where you want to import the Bunyan Logging: 

    // Import logger implementation
import {BunyanLogger, logger, setLogger} from '@raincatcher/logger';

// Instaniate logger
setLogger(new BunyanLogger({name: 'index', level:'info' }));

  2. To log a comment, add the following code after the code directly above: 

    logger.info('This log will render with BunyanLogger');

Related Information

  • For more information, see Logging Reference Information

4.1.2.2. Setting the Bunyan Logging Level

Determine the logging level you want (for more information, see Logging Reference Information) and set the Bunyan logging level accordingly.

Prerequisites

  • The Bunyan Logging Module is imported. For more information, see Importing the Bunyan Logging Module

Procedure

  1. Directly underneath the code for importing the Bunyan Logging Module, set the <LogLevel>: 

    setLogger(new ConsoleLogger(<LogLevel>));

#Example:
setLogger(new ConsoleLogger(LogLevel.INFO));

Related Information

  • For more information, see Logging Reference Information

4.1.2.3. Importing the Console Logging Module

To incorporate the Console logging into a project, the RainCatcher Logging Module is imported and then the Console Logging Module is initialized.

Prerequisites

  • raincatcher-core module must be installed.

Procedure

  1. Add the following code to the file where you want to import the Console Logging: 

    // Import logger implementation
import {ConsoleLogger, logger, setLogger} from '@raincatcher/logger';

// Instaniate logger
const log = new ConsoleLogger();;
setLogger(log);

  2. To log a comment, add the following code after the code directly above: 

    logger.info('This log will render with ConsoleLogger');

Related Information

  • For more information, see Logging Reference Information

4.1.2.4. Setting the Console Logging Level

Determine the logging level you want (for more information, see Logging Reference Information) and set the Console logging level accordingly.

Prerequisites

  • The Console Logging Module is imported. For more information, see Importing the Console Logging Module

Procedure

  1. Directly underneath the code for importing the Console Logging Module, set the <LogLevel>: 

    setLogger(new BunyanLogger(<LogLevel>));

#Example:
setLogger(new BunyanLogger(LogLevel.INFO));

Related Information

  • For more information, see Logging Reference Information
